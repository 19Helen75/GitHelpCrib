# Добро пожаловать в Гит

## Базовые команды

### Команды в консоли Git Bash

---

#### Навигация

| `Команда` | Описание |
| --- | --- |
| `pwd` | Показать текущий каталог |
| `ls` | Показать файлы в данном формате, кроме скрытых |
| `ls -f` | Показать файлы в данном примере, и скрытые включения |
| `cd c:/` | Перейти в конкретный каталог |
| `cd - ` | Вернуться назад |
| `cd ..` | Выйти на 1 уровень вверх |
| `cd ../..` | Выйти на 2 вверх уровня |
---

##### Создание каталогов

| `Команда` | Описание |
| --- | --- |
| `mkdir` | — Создать ссылку |
| `cd !$` | — Перейти только к созданной ссылке |
| `mkdir -p {app1,app2}` | Создать сразу несколько папок |
| `mkdir -p app/{css,js}` | Создать сразу несколько вложенных папок |

---

##### Создание файлов

| `Команда` | Описание |
| --- | --- |
| `touch <имя файла>` | Создать файл  |
| `touch app/{css/main.css,js/main.js,index.html}` | Создать сразу несколько файлов, лишних пробелов быть не должно |

---

##### Удаление файлов

| `Команда` | Описание |
| --- | --- |
| `rm <имя файла>` | Удалить файл |

---

##### Перемещение файлов

| `Команда` | Описание |
| --- | --- |
| `mv app1/*.* app2` | Переместить все файлы из папок app1 в адрес app2 |

---

##### Удаление директорий

| `Команда` | Описание |
| --- | --- |
| `rmdir <имя директории>` | удалить папку |
| `rm -r <имя директории>`` | удалить папку и всё, что она содержит |

---

##### Полезные возможности

Команды необязательно печатать и выполнять по очереди. Можно указать их списком — разделить двумя амперсандами (&&).
У консоли есть собственная память — буфер с несколькими последними командами. 
По ним можно перемещаться с помощью клавиш со стрелками вверх  и вниз.


Чтобы не вводить название файла или папки полностью, можно набрать первые символы имени и дважды нажать Tab. Если файл или папка есть в текущей директории, командная строка допишет путь сама.

### GitHarb


| `Команда` | Описание |
| --- | --- |
| `git init ` | Инициализировать репозиторий |
| `удалить скрытую подпапку .git ` | Если вы ошиблись и случайно инициализировали не ту папку |
| `git status ` | Проверить статус, или состояние, репозитория |
| `git add <имя файла>` | позволяет подготовить файл к сохранению например |
| `git add --all` | подготовит к сохранению сразу все файлы  |
| `git add .` | добавить в репозиторий текущую папку со всеми файлами |
| `git commit` | создать комит |
| `git commit -m "комментарий"` | создать комит, Ключ -m позволяет присвоить коммиту сообщение |
| `git log` | посмотреть коммиты |


GitHub — платформа, которая работает с Git и упрощает командное взаимодействие.

`SSH` — протокол, который обеспечивает безопасный обмен данными в сети и использует для этого ключи.


`SSH-ключ` — ваш виртуальный идентификатор в GitHub. Как ключ от квартиры, он позволяет получить доступ к GitHub-репозиторию. Также `SSH` используется для доступа к другим удалённым серверам.


`SSH-ключ` состоит из двух частей — публичной и приватной. Публичный ключ зашифрует данные, а приватный — расшифрует. Приватным ключом ни в коем случае нельзя делиться, иначе любой сможет расшифровать все ваши секреты!
